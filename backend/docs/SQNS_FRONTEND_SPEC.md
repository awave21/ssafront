# Техническое задание для фронтенда: SQNS Smart Integration

## 🎯 Цель

Добавить UI для управления SQNS интеграцией с кэшированием данных и настройкой услуг/категорий для агента.

---

## 🎨 UI/UX Концепция

### Общая структура

Страница настроек SQNS агента должна содержать 3 основные секции:

1. **Статус интеграции** (header с индикаторами и warning)
2. **Управление услугами** (таблица с фильтрами)
3. **Управление категориями** (cards/список)

### 2. Кнопка "Синхронизировать" (ОБЯЗАТЕЛЬНО)

**Расположение:** Header секции SQNS интеграции, справа от статуса

**Визуал:**

```
┌────────────────────────────────────┐
│ 🟢 SQNS активна                    │
│ Синхронизировано: 28.01 12:34     │
│                  [🔄 Обновить]    │ ← Эта кнопка
└────────────────────────────────────┘
```

**API Endpoint:**

```
POST /api/agents/{agent_id}/sqns/sync
```

**Response:**

```typescript
interface SyncResult {
  success: boolean;
  message: string;
  resources_synced: number;
  services_synced: number;
  categories_synced: number;
  links_synced: number;
  synced_at: string;
}
```

**UI состояния:**

1. **Default:**
   - Текст: "Обновить данные"
   - Иконка: 🔄 (refresh)
   - Тип: secondary button

2. **Loading:**
   - Текст: "Синхронизация..."
   - Иконка: spinner (вращается)
   - Button disabled

3. **Success:**
   - Toast (success):
     ```
     ✅ Синхронизация завершена
     Обновлено: 247 услуг, 15 специалистов, 8 категорий
     ```
   - Обновить timestamp в header

4. **Error:**
   - Toast (error):
     ```
     ❌ Ошибка синхронизации
     {error.message}
     ```

## 🎨 Дизайн система

**Статусы:**

- ✅ Включено: `green.500` (#10B981)
- ❌ Отключено: `gray.400` (#9CA3AF)
- ⚠️ Warning: `yellow.500` (#F59E0B)
- 🔴 Error: `red.500` (#EF4444)

**Приоритеты:**

```typescript
const getPriorityBadge = (priority: number) => {
  if (priority >= 10) return { color: "green", label: "Высокий" };
  if (priority >= 5) return { color: "yellow", label: "Средний" };
  return { color: "gray", label: "Низкий" };
};
```

### Компоненты

**Используемые UI компоненты:**

- `Table` - для списка услуг
- `Card` - для категорий
- `Badge` - для категорий и приоритетов
- `Toggle/Switch` - для is_enabled
- `Input` - для поиска и приоритета
- `Select/Dropdown` - для фильтров
- `Modal/Dialog` - для подтверждений
- `Toast` - для уведомлений
- `Alert` - для sqns_warning

## 🔧 Технические детали

### State management

```typescript
interface SQNSState {
  // Общий статус
  isLoading: boolean;
  isSyncing: boolean;
  lastSyncAt: string | null;
  warning: string | null;

  // Управление услугами
  services: {
    data: Service[];
    total: number;
    filters: {
      search: string;
      category: string | null;
      is_enabled: boolean | null;
    };
    pagination: {
      limit: number;
      offset: number;
    };
    isLoading: boolean;
    selectedIds: string[];
    isBulkUpdating: boolean;
  };

  // Категории
  categories: {
    data: Category[];
    isLoading: boolean;
  };

  // Preview удаления
  disablePreview: {
    data: DisablePreview | null;
    isLoading: boolean;
  };
}
```

### API Error handling

```typescript
const handleError = (error: ApiError) => {
  const messages = {
    404: "Данные не найдены",
    400: `Некорректные данные: ${error.detail}`,
    502: "Ошибка подключения к SQNS. Попробуйте позже",
    500: "Ошибка сервера. Обратитесь к администратору",
  };

  toast.error(messages[error.status] || "Неизвестная ошибка");
};
```

### Performance оптимизации

1. **Debounce поиска:** 300ms
2. **Мемоизация списков:** React.memo для строк таблицы
3. **Виртуализация:** если услуг > 100 (react-window)
4. **Optimistic updates:** для toggle/priority (откат при ошибке)
5. **React Query:**
   ```typescript
   const { data, isLoading } = useQuery(
     ["services", agentId, filters],
     () => fetchServices(agentId, filters),
     { staleTime: 60000 }, // 1 минута
   );
   ```
