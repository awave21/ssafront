# Формулы метрик аналитики (MVP)

Документ фиксирует, как сейчас считаются метрики в разделе `/analytics` и на каких данных они основаны.

## Источники данных

- Визиты: `sqns_visits`
- Платежи: `sqns_payments`
- Клиенты: `sqns_clients`
- Каналы: `sqns_visits.raw_data` + `channels/agent_channels` (для справочника фильтров)
- Теги клиента: `sqns_clients.tags`

## Период и фильтры

- Все метрики считаются в рамках выбранного периода (`date_from`, `date_to`) и таймзоны.
- Фильтр `channel` применяется к визитам по каналу из `raw_data`.
- Фильтр `client_tags` применяется по пересечению с `sqns_clients.tags`.
- Платежи при фильтрах каналов/тегов учитываются только если они привязаны к отфильтрованным визитам (`visit_external_id`) или клиенту с нужным тегом.

## Базовые определения

### Записи

- Все визиты в периоде (`sqns_visits`) без удаленных записей.

### Дошедшие

- Визит считается дошедшим, если:
  - `attendance > 0`, или
  - в `raw_data.status` содержится маркер завершенного/дошедшего визита.
- Отмененные/no-show статусы исключаются.

### Первичный визит

Приоритет определения:

1. Если в `sqns_visits.is_primary_visit` уже установлен флаг (после синхронизации) — используется он.
2. Если флаг не заполнен (переходный период) — явная метка в визите (`raw_data`) через поля вида `isPrimary/primary/firstVisit/...` или текстовые поля `type/visitType/...` со значениями типа `primary/первичный`.
3. Если явной метки нет — fallback по истории:
   - берется самый ранний визит клиента (`MIN(visit_datetime)` по `client_external_id`);
   - если текущий визит совпадает с первым по времени (с допуском 60 секунд), он считается первичным.
4. Если история недоступна — fallback по профилю клиента (`sqns_clients.client_type`, `visits_count`).

### Повторный визит

Приоритет определения:

1. Явная метка в визите (`isRepeat/repeat/...` или текстовые маркеры `repeat/повторный`).
2. Если явной метки нет — fallback по истории:
   - визит считается повторным, если он позже первого визита клиента.
3. Если история недоступна — fallback по профилю клиента (`client_type/visits_count`).

### Пациент в базе

- Пациент считается «в базе» для выбранного периода, если его первый визит был **раньше начала периода**.
- Если данных о первом визите нет, используется fallback по типу клиента/признаку повторного визита.

## Формулы KPI

- `Записи` = `COUNT(визитов в периоде)`
- `Дошедшие всего` = `COUNT(визитов, где дошел=true)`
- `Первичные записи` = `COUNT(визитов, где первичный=true)`
- `Первичные дошедшие` = `COUNT(визитов, где первичный=true AND дошел=true)`
- `Конверсия первичных` = `Первичные дошедшие / Первичные записи * 100`
- `Дошедшие первичные` = `COUNT(визитов, где дошел=true AND первичный=true)`
- `Повторные` = `COUNT(визитов, где повторный=true)`
- `Записи от первичных` = `COUNT(визитов, где первичный=true)`
- `Записи от пациентов в базе` = `COUNT(визитов, где пациент_в_базе=true)`
- `% дошедшие/записанные` = `Дошедшие всего / Записи * 100`
- `Объем выручки` = `SUM(sqns_payments.amount)`
- `Средний чек` = `SUM(amount) / COUNT(payments)`
- `Платежи` = `COUNT(платежей с amount)`
