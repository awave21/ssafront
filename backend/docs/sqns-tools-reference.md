# SQNS Tools Reference - Справочник инструментов SQNS CRM

Этот документ описывает все доступные инструменты для работы с SQNS CRM через MCP (Model Context Protocol) интеграцию.

## Оглавление

1. [sqns_list_resources](#sqns_list_resources) - Список специалистов
2. [sqns_list_services](#sqns_list_services) - Список услуг
3. [sqns_find_client](#sqns_find_client) - Поиск клиента по телефону
4. [sqns_list_slots](#sqns_list_slots) - Свободные слоты для записи
5. [sqns_list_visits](#sqns_list_visits) - Список записей (визитов)
6. [sqns_create_visit](#sqns_create_visit) - Создание новой записи
7. [sqns_update_visit](#sqns_update_visit) - Обновление записи
8. [sqns_delete_visit](#sqns_delete_visit) - Отмена записи

## sqns_list_resources

**Описание:** Получить список всех доступных сотрудников клиники (врачей, специалистов).

**Когда использовать:**

- Клиент спрашивает "какие специалисты у вас работают?"
- Нужно узнать, кто ведет прием
- Перед созданием записи нужно выбрать специалиста

**Параметры:** Нет параметров (просто вызови инструмент)

**Возвращает:**

```json
[
  {
    "id": 15,
    "name": "Иванов Иван Иванович",
    "specialization": "Стоматолог-терапевт",
    "schedule": {
      "monday": "09:00-18:00",
      "tuesday": "09:00-18:00"
    }
  },
  {
    "id": 23,
    "name": "Петрова Анна Сергеевна",
    "specialization": "Стоматолог-ортодонт",
    "schedule": {
      "wednesday": "10:00-19:00",
      "friday": "10:00-19:00"
    }
  }
]
```

**Пример использования AI агентом:**

```
Пользователь: "Какие специалисты у вас работают?"

Агент вызывает: sqns_list_resources()

Агент отвечает: "У нас работают следующие специалисты:
1. Иванов Иван Иванович - Стоматолог-терапевт (пн-вт, 09:00-18:00)
2. Петрова Анна Сергеевна - Стоматолог-ортодонт (ср, пт, 10:00-19:00)"
```

## sqns_list_services

**Описание:** Получить полный перечень медицинских услуг клиники.

**Когда использовать:**

- Клиент спрашивает "какие услуги вы предоставляете?"
- Нужно узнать стоимость и длительность процедуры
- Перед созданием записи нужно выбрать услугу

**Параметры:** Нет параметров

**Возвращает:**

```json
[
  {
    "id": 123,
    "name": "Консультация стоматолога",
    "duration": 30,
    "price": 1500,
    "category": "Терапия"
  },
  {
    "id": 456,
    "name": "Лечение кариеса",
    "duration": 60,
    "price": 3500,
    "category": "Терапия"
  }
]
```

**Пример использования:**

```
Пользователь: "Сколько стоит консультация?"

Агент вызывает: sqns_list_services()

Агент отвечает: "Консультация стоматолога стоит 1500 рублей, длится 30 минут."
```

## sqns_find_client

**Описание:** Найти клиента в базе по номеру телефона.

**Когда использовать:**

- Перед созданием новой записи (чтобы не создавать дубликаты)
- Для просмотра истории визитов клиента
- Для получения ФИО/контактов клиента (создание визита делается по телефону)

**Параметры:**

- `phone` (string, обязательный): Телефон в любом формате
  - Примеры: "+79001234567", "8-900-123-45-67", "9001234567"
  - Система автоматически нормализует формат

**Возвращает:**

```json
{
  "client": {
    "id": 789,
    "name": "Сидоров Петр Васильевич",
    "phone": "79001234567",
    "email": "sidorov@example.com",
    "previous_visits": 3
  }
}
```

Если клиент не найден:

```json
{
  "client": null
}
```

**Пример использования:**

```
Пользователь: "Запишите меня на прием, мой телефон +7-900-123-45-67"

Агент вызывает: sqns_find_client(phone="+7-900-123-45-67")

Если найден: "Здравствуйте, Петр Васильевич! Вижу, что вы уже были у нас 3 раза."
Если не найден: "Не нашел вас в базе. Как я могу к вам обращаться?"
```

## sqns_list_slots

**Описание:** Получить список свободных временных слотов для записи.

**Когда использовать:**

- Клиент хочет записаться на прием
- Нужно показать доступное время
- Перед созданием записи нужно проверить, что слот свободен

**Параметры:**

- `resource_id` (integer, обязательный): ID специалиста из sqns_list_resources
- `date` (string, обязательный): Дата в формате YYYY-MM-DD (например, "2026-01-25")
- `service_ids` (array of integers, опциональный): Список ID услуг для фильтрации

**Возвращает:**

```json
{
  "resource": {
    "id": 15,
    "name": "Иванов Иван Иванович"
  },
  "date": "2026-01-25",
  "slots": [
    {
      "datetime": "2026-01-25T09:00:00",
      "isAvailable": true,
      "duration": 30
    },
    {
      "datetime": "2026-01-25T09:30:00",
      "isAvailable": true,
      "duration": 30
    },
    {
      "datetime": "2026-01-25T10:00:00",
      "isAvailable": false,
      "reason": "Занято"
    }
  ]
}
```

**Пример использования:**

```
Пользователь: "Какое время свободно у Иванова на завтра?"

Агент вызывает:
1. sqns_list_resources() - получить resource_id Иванова (15)
2. sqns_list_slots(resource_id=15, date="2026-01-26")

Агент отвечает: "У Иванова завтра свободно:
- 09:00
- 09:30
- 11:00
- 14:00"
```

## sqns_list_visits

**Описание:** Получить список всех записей (визитов) за указанный период.

**Когда использовать:**

- Просмотр расписания клиники
- Проверка существующих записей клиента
- Поиск visit_id для обновления или отмены

**Параметры:**

- `date_from` (string, обязательный): Начало периода в формате YYYY-MM-DD
- `date_till` (string, обязательный): Конец периода в формате YYYY-MM-DD
- `per_page` (integer, опциональный): Количество записей (по умолчанию 100)

**Возвращает:**

```json
{
  "visits": [
    {
      "id": 1001,
      "client": {
        "id": 789,
        "name": "Сидоров Петр Васильевич",
        "phone": "79001234567"
      },
      "resource": {
        "id": 15,
        "name": "Иванов Иван Иванович"
      },
      "service": {
        "id": 123,
        "name": "Консультация стоматолога"
      },
      "datetime": "2026-01-25T14:00:00",
      "status": "confirmed",
      "comment": "Первичный прием"
    }
  ]
}
```

**Пример использования:**

```
Пользователь: "Покажите мои записи на эту неделю"

Агент вызывает: sqns_list_visits(date_from="2026-01-25", date_till="2026-01-31")

Агент отвечает: "У вас 1 запись:
- 25 января в 14:00 к Иванову И.И. - Консультация стоматолога"
```

## sqns_create_visit

**Описание:** Создать новую запись на прием в SQNS CRM.

**Когда использовать:**

- Клиент хочет записаться на прием
- После того как выбрали специалиста, услугу и время

**ВАЖНО: Перед созданием записи всегда:**

1. Найди клиента через `sqns_find_client` (по телефону) — чтобы не создавать дубликаты и получить ФИО/контакты
2. Получи список специалистов через `sqns_list_resources`
3. Получи список услуг через `sqns_list_services`
4. Проверь свободные слоты через `sqns_list_slots`

**Параметры:**

- `resource_id` (integer, обязательный): ID специалиста из sqns_list_resources
- `service_id` (integer, обязательный): ID услуги из sqns_list_services
- `datetime` (string, обязательный): Дата и время в формате ISO 8601
  - Пример: "2026-01-25T14:30:00"
  - Выбирай только из свободных слотов!
- `user_name` (string, обязательный): ФИО клиента (из `sqns_find_client.client.name` или от пользователя)
- `phone` (string, обязательный): Телефон клиента (из `sqns_find_client.client.phone` или от пользователя)
- `email` (string, опциональный): Email клиента
- `comment` (string, опциональный): Комментарий к записи

**Возвращает:**

```json
{
  "visit": {
    "id": 1002,
    "resource_id": 15,
    "service_id": 123,
    "datetime": "2026-01-25T14:30:00",
    "status": "confirmed",
    "comment": "Первичный прием",
    "created_at": "2026-01-24T12:00:00"
  }
}
```

**Пример использования:**

```
Пользователь: "Запишите меня к Иванову на завтра в 14:30, телефон +79001234567"

Агент выполняет:
1. client = sqns_find_client(phone="+79001234567")
2. resources = sqns_list_resources() → находит Иванова, resource_id=15
3. services = sqns_list_services() → выбирает услугу, service_id=123
4. slots = sqns_list_slots(resource_id=15, date="2026-01-26")
   → проверяет, что 14:30 свободно
5. visit = sqns_create_visit(
     resource_id=15,
     service_id=123,
     datetime="2026-01-26T14:30:00",
     user_name=client["client"]["name"],
     phone=client["client"]["phone"],
     comment="Запись через чат-бота"
   )

Агент отвечает: "Отлично! Я записал вас к Иванову И.И. на 26 января в 14:30.
Консультация стоматолога, длительность 30 минут. Номер записи: 1002."
```

## sqns_update_visit

**Описание:** Обновить существующую запись на прием.

**Когда использовать:**

- Клиент хочет перенести запись на другое время
- Нужно добавить комментарий к записи
- Изменить статус визита

**Параметры:**

- `visit_id` (integer, обязательный): ID записи из sqns_list_visits или sqns_create_visit
- `datetime` (string, опциональный): Новая дата и время в формате ISO 8601
- `comment` (string, опциональный): Новый комментарий
- `status` (string, опциональный): Новый статус ("confirmed", "cancelled", "completed")

**Возвращает:**

```json
{
  "visit": {
    "id": 1002,
    "resource_id": 15,
    "service_id": 123,
    "datetime": "2026-01-26T15:00:00",
    "status": "confirmed",
    "comment": "Перенесено по просьбе клиента",
    "updated_at": "2026-01-24T13:00:00"
  }
}
```

**Пример использования:**

```
Пользователь: "Можно перенести мою запись на 15:00?"

Агент выполняет:
1. visits = sqns_list_visits(date_from="2026-01-25", date_till="2026-01-31")
   → находит запись клиента, visit_id=1002
2. slots = sqns_list_slots(resource_id=15, date="2026-01-26")
   → проверяет, что 15:00 свободно
3. sqns_update_visit(
     visit_id=1002,
     datetime="2026-01-26T15:00:00",
     comment="Перенесено по просьбе клиента"
   )

Агент отвечает: "Готово! Я перенес вашу запись на 26 января в 15:00."
```

## sqns_delete_visit

**Описание:** Удалить (отменить) запись на прием.

**Когда использовать:**

- Клиент хочет отменить запись
- Запись была создана по ошибке

**⚠️ ВНИМАНИЕ:** Это действие необратимо - запись будет полностью удалена из системы!

**Параметры:**

- `visit_id` (integer, обязательный): ID записи из sqns_list_visits или sqns_create_visit

**Возвращает:** None (ничего - успешное удаление не возвращает данных)

Примечание: при валидационных ошибках SQNS (4xx) инструмент может вернуть объект ошибки вида  
`{ "ok": false, "retryable": false, "error": { "message": "...", "status_code": 4xx } }` — в этом случае нужно
показать ошибку пользователю и уточнить данные, а не повторять вызов.

**Пример использования:**

```
Пользователь: "Отмените мою запись на завтра"

Агент выполняет:
1. visits = sqns_list_visits(date_from="2026-01-26", date_till="2026-01-26")
   → находит запись клиента, visit_id=1002
2. sqns_delete_visit(visit_id=1002)

Агент отвечает: "Ваша запись на 26 января в 14:30 отменена."
```

## Типичные сценарии использования

### Сценарий 1: "Какие специалисты работают?"

```
1. sqns_list_resources()
2. Показать список специалистов с их специализацией
```

### Сценарий 2: "Запишите меня на прием"

```
1. sqns_find_client(phone="...") → получить ФИО/контакты (или понять, что клиента нет)
2. sqns_list_resources() → выбрать специалиста
3. sqns_list_services() → выбрать услугу
4. sqns_list_slots(resource_id=X, date="...") → показать свободное время
5. sqns_create_visit(resource_id=Y, service_id=Z, datetime="...", user_name="...", phone="...", comment?=...)
```

### Сценарий 3: "Перенесите мою запись"

```
1. sqns_list_visits(date_from="...", date_till="...") → найти visit_id
2. sqns_list_slots(resource_id=X, date="...") → проверить новое время
3. sqns_update_visit(visit_id=X, datetime="...")
```

### Сценарий 4: "Отмените мою запись"

```
1. sqns_list_visits(date_from="...", date_till="...") → найти visit_id
2. sqns_delete_visit(visit_id=X)
```

## Рекомендации для AI агента

### ✅ Правильно:

1. **Всегда используй актуальные данные из инструментов**, а не придумывай информацию
2. **Проверяй свободные слоты** перед созданием записи
3. **Находи клиента** через sqns_find_client перед созданием записи
4. **Подтверждай действия** с клиентом перед удалением записи
5. **Показывай детали**: имя специалиста, дата, время, название услуги

### ❌ Неправильно:

1. ❌ "У нас работают разные специалисты" - вместо вызова sqns_list_resources
2. ❌ Создавать запись без проверки свободных слотов
3. ❌ Создавать дубликаты клиентов вместо поиска по телефону
4. ❌ Удалять записи без подтверждения клиента
5. ❌ Использовать неправильный формат даты (всегда ISO 8601!)

## Форматы данных

### Дата и время

- **Дата:** `YYYY-MM-DD` (например, `2026-01-25`)
- **Дата и время:** `YYYY-MM-DDTHH:MM:SS` (например, `2026-01-25T14:30:00`)

### Телефон

- Любой формат: `+79001234567`, `8-900-123-45-67`, `9001234567`
- Система автоматически нормализует

### Статусы визитов

- `confirmed` - подтверждено
- `cancelled` - отменено
- `completed` - выполнено
- `no_show` - не явился

---

## Обработка ошибок

Если инструмент возвращает ошибку:

1. **Токен истек**: Попроси администратора переподключить SQNS
2. **Слот занят**: Предложи клиенту другое время
3. **Клиент не найден**: Создай нового клиента или попроси уточнить телефон
4. **Неверный формат даты**: Проверь формат (должен быть ISO 8601)

## Дополнительная информация

- Все инструменты работают через MCP (Model Context Protocol)
- Аутентификация происходит автоматически через настроенный SQNS credential
- Все запросы логируются для аудита
- Ошибки автоматически обрабатываются и логируются

Для настройки SQNS интеграции см. [sqns-frontend-spec.md](./sqns-frontend-spec.md)
